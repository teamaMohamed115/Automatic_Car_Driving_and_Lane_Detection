# 4WD Autonomous Robot Car - Implementation Plan

## Overview
We are developing an autonomous lane-following robot car on a Raspberry Pi 4 Model B running Ubuntu 24.04 (ARM64). 

## Primary Coding Constraints
- **Architecture:** Must support ARM64.
- **Camera Access:** Use `libcamerasrc` via GStreamer for all video streams. DO NOT use `raspistill` or `raspivid` (deprecated).
- **Environment:** Code is developed on x86_64 laptops (simulation/testing) and deployed to ARM64 Raspberry Pi (production).
- **Hardware Abstraction:** Implement a `HardwareInterface` class to handle environment differences (e.g., simulated camera on laptop, physical CSI camera on Pi).

## Development Roadmap

### Phase 1: Infrastructure & Abstraction
- Implement `CameraManager` class.
  - Pi Path: `cv2.VideoCapture("libcamerasrc ! ... ! appsink", cv2.CAP_GSTREAMER)`
  - Laptop Path: Read from local MP4 file or webcam.
- Implement `MotorController` class.
  - Pi Path: Use `gpiozero` or `lgpio`.
  - Laptop Path: Dummy print statements (simulate motion).

### Phase 2: Vision Pipeline
- Utilize the `nickpai/lane-detection-unet-ncnn` model.
- Implement the model wrapper to handle NCNN inference on the Pi.
- Process input frames:
  1. Capture frame.
  2. Pre-process (Resize/Normalization for the NCNN model).
  3. Run inference.
  4. Extract lane coordinates.

### Phase 3: Control Logic
- Convert lane coordinates to steering angle (PID controller or simple error-based logic).
- Map steering angle to differential motor speed (Left/Right motor speeds).

### Phase 4: Integration
- Combine Vision and Motor modules.
- Add safety kill-switch (stop car if lane confidence is low).
